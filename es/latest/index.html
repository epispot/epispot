Paquete epispot

épispot v2

Un paquete para el modelado matemático de enfermedades infecciosas. Originalmente diseñado para los epidemiólogos, épispot puede cambiarse para casi cada tipo de modelado.

Características

Actualmente el paquete épispot solamente es compatible con modelos compartimentales, aunque planeamos a expandir el paquete para ser compatible con modelos estocásticos y espaciales también. Actualmente, épispot ofrece lo siguiente:

  • La compilación rápida de modelos compartimentales con los compartimientos:
    • Susceptible
    • Infectado
    • Recuperado
    • Eliminado
    • Expuesto
    • Muerto
    • Crítico
    • Hospitalizado
  • Los compartimientos definidos a medida para la investigación
  • El creador incorporado de gráficos y visualización
    • Interactivamente traza las predicciones del modelo
    • Crea comparaciones entre modelos

A causa de su gama diverso de características, épispot puede usarse para la investigación y el modelado experimental. Si querrías agregar más compatibilidad para el modelado, vea la sección de contribuir.

Instalación

El paquete épispot puede instalarse de PyPI, Anaconda, o construirse desde la fuente. Antes de leer esta guía, es importante anotar que realmente hay «dos» versiones diferentes del paquete épispot. El primero es el paquete desde la rama master, que siempre tendrá una etiqueta de versión como v#.#.#. Este paquete se usa para lanzar versiones estables de épispot. Sin embargo, durante eventos importantes, el paquete desde la rama nightly se usa para lanzar unas características rápidamente. Este paquete se denomina como la versión nocturna. Sin embargo, estas versiones podrían ser inestables.

PyPI

Esta es la manera más fácil de instalar épispot. Desde un terminal, teclee:

pip install epispot

Para la versión nocturna, use

pip install epispot-nightly

Pip pedirá que instale numpy y matplotlib como dependencias si no los ha instalado todavía. Además, podría requerir que instale fire por la interfaz de línea de comandos. Estes paquetes pueden instalarse antes con:

pip install numpy
pip install matplotlib
pip install fire

Anaconda

Por favor observe que la versión nocturna no está disponible en el índice de paquetes de Anaconda. Sin embargo, se puede instalarse en sistemas que usan conda con

pip install epispot-nightly

que usa pip de Anaconda para instalarlo.

La versión estándar de épispot se lanza a conda usando el canal conda-forge. Para instalarlo, use:

conda config --add channels conda-forge
conda install -c conda-forge epispot

Construir desde la fuente

Esta es la manera más difícil de instalar <a title="epispot" href="#epispot">epispot</a> y es aconsejable que use PyPI o Anaconda para instalarlo en vez de eso. Sin embargo, si querrías contribuir al repositorio, esto será especialmente útil.

Clone el repositorio con:

git clone https://github.com/epispot/epispot  # clonar epispot/epispot
cd epispot  # abrir proyecto
pip install -r requirements.txt  # instalar requisitos del paquete
pip install -r bin/requirements.txt  # instalar requisitos de la interfaz de línea de comandos

Entonces, construya la versión nocturna con

python setup-nightly.py install

Para una versión estable, salte a una rama de lanzamiento con algo como:

git checkout v2.1.1
python setup.py install

Documentación

La documentación para el paquete épispot se genera automáticamente desde el código usando Pdoc3. Se puede ver la documentación para las dos versiones de épispot aquí.

Al principio, la documentación puede parecer un poco difícil de entender, especialmente si no se tiene mucha experiencia con la epidemiología. Por eso épispot ha creado un manual entero que describe unos conceptos básicos y necesarios para entender épispot. Se puede verlo aquí. La fuente desde GitHub es disponible aquí.

Si Ud. planea a usar épispot en su proyecto, por favor cumpla con la licencia GPLv3. Esta requiere que algunos cambios que hace a épispot necesiten ser de código abierto bajo esta misma licencia. Además, requiere que le dé crédito al autor, que puede hacer citando el proyecto en su investigación, enlazando al repositorio original, o mencionando el autor @quantum9innovation.

Las menciones

Idea e Inspiración

La idea original de épispot vino de un video de 3Blue1Brown sobre la dinámica de enfermedades infecciosas y un artículo interactivo en «The Washington Post». Esto era clave a la dinámica básica de enfermedades infecciosas aquí. Sin embargo, lo que finalmente ultimó el paquete era una serie de artículos por Henry Froese, disponible en Medium aquí, con sus cuadernos interactivos correspondientes.

Desarrollo y mantenimiento del código

El proyecto épispot es construido sobre otro código abierto y es sí mismo de código abierto. El desarrollo inicial se hizo posible por @quantum9innovation y @Quantalabs ocupó del mantenimiento de una gran parte del código. Además, ¡gracias a todos de los contribuidores de código abierto de épispot!

Dependencias

El equipo de épispot también usa los siguientes proyectos como dependencias:

  • NumPy (GitHub), el paquete fundamental para la computación científica con Python
  • Matplotlib (GitHub), trazar con Python
  • Google Fire (Github), un paquete para automáticamente generar las interfaces de línea de comandos (CLIs) de absolutamente algún objeto de Python.
